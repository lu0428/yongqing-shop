# 團購網頁系統重建流程
> 這份文件讓你在需要從頭建立一個新客戶（B端）的系統時，照步驟做就能完成

---

## 系統組成概覽

```
GitHub Pages（前端網頁）
    ↕ API 呼叫
Google Apps Script - B端（後端邏輯）
    ↕ 讀取資料
Google Sheets - B端（訂單 + 商品 + 團購主設定）
    ↕ 授權驗證（直接讀 Sheet）
Google Sheets - A端（授權管理）← 你自己的，一份就好，不需重建
```

---

## 重建前的準備

- [ ] 一個 Google 帳號（部署 GAS 和 Google Sheet 用）
- [ ] 一個 GitHub 帳號（放前端網頁用）
- [ ] 本地備份的 `index.html` 和 `訂單後台_Apps_Script.gs`

---

## STEP 1：建立 B端 Google Sheet

1. 開啟 [Google Sheets](https://sheets.google.com) → 建立新的試算表
2. 命名：`詠晴漢方訂單系統`（或客戶名稱）
3. 建立以下三張工作表（點左下 + 新增）：

### 工作表一：「訂單」
- 第一列標題（從 A1 開始）：
  `時間 | 姓名 | 手機 | 地址 | 付款方式 | 商品明細 | 總金額 | 備註 | 團購主 | 確認是否收款 | 訂單狀況`
- ※ 實際上 GAS 會自動建立這張表，可以不用手動建

### 工作表二：「團購主商品管理」
- 第一列標題：
  `商品名稱 | 分類 | 價格 | 規格 | 描述 | 圖片路徑 | 標籤 | 庫存 | 單份標籤 | 組合價 | 組合描述 | 多圖片路徑 | [C端1名稱] | [C端2名稱] ...`
- 從第二列起逐筆填入商品資料
- 「庫存」欄：插入 → 勾選框（格式 → 勾選框）
- 各 C端欄位：同樣用勾選框，TRUE = 該 C端可賣此商品

### 工作表三：「團購主設定」
- 第一列標題：
  `URL參數 | 顯示名稱 | 是否啟用 | 密碼`
- 範例資料：
  ```
  junhsu | JunHsu | TRUE | （密碼或空白）
  shuntian | 順天宮 | TRUE | abc123
  ```
- 「是否啟用」欄：用勾選框

## STEP 2：部署 B端 GAS

1. 在上面建的 Google Sheet → 點「擴充功能」→「Apps Script」
2. 把 `訂單後台_Apps_Script.gs` 的所有程式碼貼上（全選取代原本的 `function myFunction(){}`）
3. 確認以下變數設定正確：
   ```javascript
   var A_SHEET_ID = '1iVcl6O77c2JtMknoYb3OR8BaxEGsVIksKyyKrRb-lMk'; // A端 Sheet ID（不變）
   var CLIENT_ID  = 'yongqing'; // 這個 B端客戶的識別碼（對應 A端授權表的 A 欄）
   ```
4. 點右上「部署」→「新增部署」：
   - 類型：網頁應用程式
   - 執行身分：**我（你的帳號）**
   - 存取權限：**所有人**
5. 點「部署」→ 複製「網頁應用程式 URL」（/exec 結尾）
6. **記下這個 URL**，下一步要用

## STEP 3：建立 GitHub Repo 並上傳前端

1. 開啟 [GitHub](https://github.com) → New repository
   - 名稱：`yongqing-shop`（或客戶英文名）
   - 設定：**Public**
   - 不要加 README
2. 開本地終端機，執行：
   ```bash
   cd C:\Users\user\OneDrive\桌面\claude\yongqing-shop
   git init
   git add index.html
   git config user.email "ki.58328@gmail.com"
   git config user.name "ki58328"
   git commit -m "init"
   git remote add origin https://github.com/ki58328/yongqing-shop.git
   git branch -M main
   git push -u origin main
   ```
3. 上傳商品圖片：
   ```bash
   git add images/
   git commit -m "add images"
   git push
   ```

---

## STEP 4：修改 index.html 填入 GAS URL

1. 用文字編輯器開啟 `index.html`
2. 找到（第 889 行附近）：
   ```javascript
   const APPS_SCRIPT_URL = 'https://script.google.com/macros/s/...舊URL.../exec';
   ```
3. 替換成 STEP 2 取得的新 URL
4. 存檔，push 到 GitHub：
   ```bash
   git add index.html
   git commit -m "update GAS URL"
   git push
   ```

---

## STEP 5：開啟 GitHub Pages

1. GitHub repo 頁面 → Settings → Pages
2. Branch 選 `main`，folder 選 `/ (root)` → Save
3. 等 1～2 分鐘，網址就會是：
   `https://[GitHub帳號].github.io/[repo名稱]/`

---

## STEP 6：在 A端授權表新增這個 B端

1. 開啟 A端 Google Sheet「網頁授權管理」→「授權管理」工作表
2. 新增一列：
   ```
   識別碼      | 公司名稱 | 到期日     | 狀態 | 備註
   yongqing   | 詠晴漢方 | 2027/04/25 | 啟用 | 第一個B端
   ```
3. 「狀態」欄請用 **下拉選單**（資料驗證：啟用/停用）

## STEP 7：測試確認

- [ ] 打開網頁 URL → 商品有出現
- [ ] 加入購物車 → 下一筆訂單 → 打開 Sheet「訂單」有看到新資料
- [ ] 輸入 `?team=junhsu` → 只出現 JunHsu 能賣的商品
- [ ] 有設密碼的 C端 → 進入時要求輸入密碼
- [ ] 在 A端 Sheet 把狀態改「停用」→ 執行 `clearLicenseCache` → 網頁顯示停用畫面
- [ ] 改回「啟用」→ 執行 `clearLicenseCache` → 網頁正常

## 日常維護操作

### 新增商品
→ 直接在 Sheet「團購主商品管理」新增一列，圖片 push 到 GitHub，立即生效

### 新增 C端（團購主）
1. Sheet「團購主設定」新增一列（填 URL參數/顯示名稱/啟用/密碼）
2. Sheet「團購主商品管理」最右邊新增一欄（填 C端的顯示名稱作欄位標題）
3. 勾選這個 C端可以賣哪些商品
4. 告知 C端他的連結：`https://...github.io/.../?team=[URL參數]`

### 停用 / 恢復 B端授權
1. A端 Sheet 改「停用」或「啟用」
2. 到 B端 GAS 執行 `clearLicenseCache()`
3. 立即生效

### 更新 GAS 程式碼後重新部署
1. GAS 編輯器 → 部署 → 管理部署 → 編輯（鉛筆圖示）→ 版本選「建立新版本」→ 部署
2. 若 URL 有變 → 更新 index.html 的 `APPS_SCRIPT_URL` → push 到 GitHub

### 新增商品圖片
1. 圖片放到 `images/中西藥局/` 資料夾（檔名 .webp）
2. `git add images/ && git commit -m "add image" && git push`
3. Sheet F欄填入路徑：`images/中西藥局/檔名.webp`

## 常見問題速查

| 問題 | 解法 |
|---|---|
| 改了授權狀態但網頁沒變 | B端 GAS 執行 `clearLicenseCache()` |
| 商品都顯示「已售完」 | Sheet「庫存」欄確認是勾選框，且已勾選 |
| 圖片不顯示 | 確認 Sheet F欄路徑與 GitHub 上的檔名完全一致（含空格） |
| 新 C端進不去 | 確認「團購主設定」Sheet 有這筆資料，且「是否啟用」=TRUE |
| 密碼每次都要重新輸入 | 應使用 localStorage（已修正），清一下瀏覽器快取試試 |
| GAS 部署後 URL 變了怎麼辦 | 更新 index.html 第 889 行的 APPS_SCRIPT_URL，push 到 GitHub |
